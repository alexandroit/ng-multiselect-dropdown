Core examples

Start with multiple selection, single selection, and reactive forms. The examples below cover the most common production setups for the component.

Multiple selection with object data

Use idField and textField to map custom objects while keeping search and select-all enabled. 

{{ basicTsCode }}
<ng-multiselect-dropdown
  name="basicCities"
  [data]="basicCities"
  [settings]="basicSettings"
  [(ngModel)]="basicSelected">
</ng-multiselect-dropdown>
{{ basicSelected | json }}

Reactive forms

Bind the component to a FormGroup with formControlName and treat it like any other Angular form control. 

{{ reactiveTsCode }}
<form [formGroup]="reactiveForm">
  <ng-multiselect-dropdown
    formControlName="cities"
    [data]="reactiveCities"
    [settings]="reactiveSettings">
  </ng-multiselect-dropdown>
</form>
{{ reactiveForm.value | json }}

Quick API reference

These are the inputs and outputs most teams reach for first when integrating the dropdown into real forms.

Core inputs

[data]Array of strings or objects used as the available options.
[settings]Behavior flags such as search, limits, and custom field mapping.
[(ngModel)]Two-way binds the selected values in template-driven forms.
formControlNameConnects the dropdown to a reactive form control.

Common settings

singleSelectionTurns the dropdown into a one-choice picker.
allowSearchFilterEnables the built-in search field.
limitSelectionCaps how many items can be selected.
itemsShowLimitLimits how many selected badges are shown in the trigger.
allowRemoteDataSearchKeeps the search box visible even with no loaded items.

Outputs

(onSelect)Emits the selected item.
(onDeSelect)Emits when a selected item is removed.
(onSelectAll)Emits the full visible list when select-all runs.
(onDeSelectAll)Emits when all selections are cleared.
(onFilterChange)Emits the current search text for remote queries.
